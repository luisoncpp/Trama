AI Import & Export

Trama offers clipboard-based project interchange for working with large language models. Export selected project files in a structured multi-file format, or import structured AI output back into your workspace with a preview step.

How to Access

Open the Import and export panel from the sidebar rail (the transfer icon). Under Project interchange you will find:

  • Import AI Content — paste structured clipboard text from an LLM and preview changes before writing files.
  • Export Files — stage files or folders, then copy them to the clipboard in Trama's structured format.

Export Workflow

  1. Click Export Files.
  2. Use Add Files or Add Folder to build a staging basket of project paths.
  3. Optionally enable Include frontmatter.
  4. Click Export & Copy. Trama writes a structured block to your clipboard, for example: 
    === FILE: lore/characters/aragon.md ===
# Aragon ...
=== FILE: lore/characters/eldric.md ===
# Eldric ...
  5. Paste the block into your AI chat for expansion, revision, or analysis.

Import Workflow

  1. Copy the prompt below and send it to your LLM together with the exported files or your instructions.
  2. Copy the model's full response to your clipboard. The response must use Trama's file header format and must not be wrapped in markdown code fences.
  3. Click Import AI Content in Trama.
  4. Paste the response into the dialog.
  5. Choose replace or append mode, then click Preview Import.
  6. Review the file list and counts, then confirm to write the changes to disk.

Prompt to send to your LLM

Copy this prompt, paste your exported Trama files after it, and send the combined message to the model:

You are helping me edit files for Trama, a file-first writing app.

Reply ONLY in Trama's multi-file interchange format. Do not add commentary before or after the files, and do not wrap the answer in markdown code fences.

Format:
=== FILE: relative/path/from/project/root.md ===
(optional YAML frontmatter)
file body here

Rules:
- Use one === FILE: ... === header per file.
- Paths must be relative to the project root, such as lore/characters/aragon.md or book/chapter-02.md.
- Keep paths stable when revising existing files.
- Preserve useful YAML frontmatter when present.
- Separate multiple files with a blank line only when needed for readability.
